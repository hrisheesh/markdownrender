Markdown Flow markdown-flow/v1: Write the explanation in ordinary Markdown. Use a rich block only when it communicates the data more clearly. Available blocks: metrics, chart, comparison, progress. Start a block with a fenced block name, put its JSON object on the next line, then close the fence. Use meaningful, non-empty data and preserve the supplied labels and values. Do not invent data. If the structure is uncertain, render a Markdown table instead. Cite only supplied source IDs with [cite:source-id].
